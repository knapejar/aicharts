# Contributing

Thanks for helping improve aicharts.

## Flow

1. Fork the repo.
2. Create a branch: `git checkout -b feat/your-change`.
3. Make your changes. Run `npm run format && npm test && npm run snapshot`.
4. Commit with a clear message.
5. Open a PR against `main`.

## New charts

Every new chart type must ship with:

- A snapshot PNG in `tests/snapshots/<chart>-*.png`, generated by `npm run snapshot`.
- At least one edge-case unit test (empty data, NaN, single point, negative values, overlap, etc.).
- A row in the capability matrix in `README.md`.
- An entry in `src/charts/index.ts` and the `chart` union in `src/core/types.ts`.

See [CLAUDE.md](./CLAUDE.md) → "How to add a new chart type" for the full checklist.

## Code style

- English only.
- No comments unless non-obvious.
- Prettier is source of truth; run `npm run format`.
- No emojis in committed files.
